# ブログ記事表示アプリ ドキュメント一覧

このディレクトリには、Markdownファイルをブログ記事として表示するアプリケーションの設計・実装に関するドキュメントが含まれています。

## ドキュメント構成

### 1. [仕様書](./blog-app-specification.md)

ブログ記事表示アプリの全体的な仕様を定義したドキュメントです。

**主な内容:**
- システム概要と目的
- 機能要件（必須機能・任意機能）
- 技術仕様（使用技術スタック）
- データモデルとAPI設計
- UI/UXデザイン
- セキュリティ・パフォーマンス・アクセシビリティ
- テスト戦略とデプロイメント

**対象読者:** プロジェクトの関係者全員（開発者、デザイナー、プロダクトマネージャー等）

### 2. [実装計画書](./blog-app-implementation-plan.md)

仕様書を基にした具体的な実装手順と計画を記載したドキュメントです。

**主な内容:**
- 3段階の実装フェーズ（Phase 1-3）
- 各フェーズの詳細なタスク一覧
- 実装手順とコード例
- テスト項目と工数見積もり
- 実装スケジュール
- デプロイ前チェックリストと保守計画

**対象読者:** 開発者

## ドキュメントの使い方

### 開発を開始する前に

1. まず **仕様書** を読んで、アプリケーション全体の設計を理解する
2. **実装計画書** を参照して、実装の進め方を確認する
3. Phase 1から順番に実装を進める

### 実装中

- 各フェーズの実装前に、該当セクションを再度確認
- テスト項目を参照しながらテストを実装・実行
- 疑問点があれば仕様書に立ち返る

### 実装後

- デプロイ前チェックリストを使用して最終確認
- 保守・運用計画を参照して運用方法を確認
